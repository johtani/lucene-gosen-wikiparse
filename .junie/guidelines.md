# プロジェクト固有のガイドライン

## 1. 文字エンコーディングと日本語の扱い

このプロジェクトでは、日本語を含むドキュメントや GitHub への投稿を行う際、文字化けを防ぐために以下のルールを適用します。

### GitHub CLI (`gh`) を使用した日本語投稿
PowerShell から `gh issue create` や `gh issue comment` を使用して日本語を投稿する場合、直接コマンド引数に文字列を渡すと文字化けする可能性があります。
以下の手順で、**BOMなし UTF-8** の一時ファイルを経由して投稿してください。

1. 日本語の内容を一時ファイル（例: `body.txt`）に保存する。
   * PowerShell の `Out-File -Encoding utf8` は BOM 付きになる場合があるため、`New-Object System.Text.UTF8Encoding($false)` を使用した .NET メソッドでの書き込みを推奨。
2. `gh` コマンドで `--body-file` 引数を使用してそのファイルを指定する。
3. 投稿後、一時ファイルを削除する。

### ファイルのエンコーディング
* ソースコード (`.java`)、設定ファイル (`build.gradle` 等)、ドキュメント (`.md`) はすべて **UTF-8 (BOMなし)** で統一します。
